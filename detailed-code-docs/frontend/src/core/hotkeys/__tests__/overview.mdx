## High-level description

The target file contains unit tests for the `parseShortcut` function, which is responsible for parsing and recognizing keyboard shortcuts. These tests ensure that the function correctly identifies valid shortcuts and rejects invalid ones by simulating keyboard events and comparing them against expected shortcut patterns.

## Code Structure

The main symbol in the code is the `parseShortcut` function, which is imported from the `shortcuts` module. The tests are organized using the `describe` and `it` functions from the `vitest` testing framework, which group and define individual test cases, respectively. Each test case simulates a keyboard event and checks if the `parseShortcut` function correctly identifies the shortcut.

### `parseShortcut`
#### Description
The `parseShortcut` function is responsible for parsing a string representation of a keyboard shortcut and returning a function that checks if a given `KeyboardEvent` matches the shortcut.

#### Inputs
| Name     | Type   | Description                        |
|:---------|:-------|:-----------------------------------|
| shortcut | string | A string representing the shortcut |

#### Outputs
| Name   | Type                      | Description                                      |
|:-------|:--------------------------|:-------------------------------------------------|
| output | (e: KeyboardEvent) =&gt; boolean | A function that checks if a `KeyboardEvent` matches the shortcut |

#### Internal Logic
The function splits the shortcut string by the "-" character to handle combined key shortcuts. It then returns a function that iterates over each part of the shortcut, checking if the corresponding key or modifier key (Ctrl, Cmd, Shift, Alt) is pressed in the `KeyboardEvent`. It also ensures that no additional modifier keys are pressed unless specified in the shortcut.

## Side Effects

There are no side effects in the `parseShortcut` function as it only processes input and returns a result without modifying any external state.
