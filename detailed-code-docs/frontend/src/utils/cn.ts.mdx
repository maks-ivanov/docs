Here's a comprehensive documentation for the `ButtonPlugin` class:

## High-level description
The `ButtonPlugin` implements a clickable button UI element for the Marimo framework. It renders a customizable button that can be clicked to increment a counter.

## Code Structure
The `ButtonPlugin` class implements the `IPlugin` interface, defining the structure and behavior of the button element. It uses the `Button` component from the UI library for rendering.

## Symbols

### ButtonPlugin
#### Description
A class that implements a clickable button UI element.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| props | IPluginProps&lt;number, Data&gt; | Props containing the button's data and value |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| JSX.Element | React component | The rendered button element |

#### Internal Logic
1. Validates the input data using a Zod schema
2. Renders a `Button` component with the specified properties
3. Handles click events to increment the button's value

### Data (interface)
Defines the structure of the button's configuration data.

| Property | Type | Description |
|:---------|:-----|:------------|
| label | string | The text displayed on the button |
| kind | Intent | The visual style of the button |
| disabled | boolean | Whether the button is disabled |
| fullWidth | boolean | Whether the button should take full width |
| tooltip | string (optional) | Tooltip text for the button |

## Dependencies
- React
- Zod for data validation
- Custom UI components (Button, Tooltip)
- Utility functions (cn, renderHTML)

## Configuration
The button's appearance and behavior can be configured through the `Data` interface properties.

## Error Handling
Input validation is handled by the Zod schema. No explicit error handling for runtime errors.

## Logging
No specific logging implemented in this plugin.

## API/Interface Reference
Implements the `IPlugin` interface with the following methods:
- `tagName`: "marimo-button"
- `validator`: Zod schema for validating input data
- `render`: Method to render the button component
