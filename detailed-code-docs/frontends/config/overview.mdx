## High-level description

This directory contains configuration files for a frontend project, primarily focusing on TypeScript and package management. It includes a `package.json` file for managing dependencies and exports, and a `tsconfig.json` file for configuring the TypeScript compiler.

## What does it do?

This configuration setup serves several purposes:

1. It centralizes and standardizes configuration for frontend projects, making it easier to maintain consistency across multiple frontend applications or components.

2. The `package.json` file defines the project's dependencies and exports, allowing other parts of the project to easily import and use shared configurations.

3. The `tsconfig.json` file sets up a modern TypeScript development environment with strict type checking and support for ES2020 features. It also configures the compiler to work with SolidJS for JSX handling.

4. It sets up Tailwind CSS with additional plugins for gradient mask images and custom scrollbars.

5. It includes Prettier for code formatting, ensuring consistent code style across the project.

## Key Files

1. `package.json`:
   - Defines the package name as "config" within a "frontends" directory.
   - Uses semantic versioning (1.0.0) for easy tracking of changes.
   - Exports various configuration files:
     - `./tsconfig`: TypeScript configuration
     - `./tailwind`: Tailwind CSS presets
     - `./prettier`: Prettier configuration
   - Lists dependencies:
     - `tailwind-gradient-mask-image` and `tailwind-scrollbar` for Tailwind CSS enhancements
     - `prettier` and `prettier-plugin-tailwindcss` for code formatting

2. `tsconfig.json`:
   - Configures the TypeScript compiler with the following key settings:
     - Targets ES2020
     - Enables strict type checking
     - Supports SolidJS for JSX handling
     - Uses the "Bundler" module resolution strategy
     - Allows importing TypeScript files with extensions
     - Enables synthetic default imports and JSON module resolution

## Dependencies

1. Tailwind CSS plugins:
   - `tailwind-gradient-mask-image`: Adds support for gradient mask images in Tailwind CSS.
   - `tailwind-scrollbar`: Provides custom scrollbar styling options for Tailwind CSS.

2. Prettier:
   - `prettier`: A code formatter for consistent style across the project.
   - `prettier-plugin-tailwindcss`: Ensures Prettier correctly formats Tailwind CSS classes.

## Configuration

The `tsconfig.json` file contains various compiler options:

1. Language and Environment:
   - `"target": "ES2020"`: Sets the ECMAScript target version.
   - `"lib": ["ES2021", "DOM", "DOM.Iterable"]`: Includes library definitions for modern JavaScript and DOM APIs.

2. Modules and Bundling:
   - `"moduleResolution": "Bundler"`: Uses the "Bundler" module resolution strategy.
   - `"allowImportingTsExtensions": true`: Allows importing TypeScript files with extensions.
   - `"resolveJsonModule": true`: Enables importing JSON files as modules.

3. JavaScript Support:
   - `"jsx": "preserve"`: Preserves JSX syntax.
   - `"jsxImportSource": "solid-js"`: Sets the JSX import source to "solid-js" for SolidJS support.

4. Linting and Type Checking:
   - `"strict": true`: Enables strict type checking.
   - `"noUnusedLocals": true`: Flags unused local variables.
   - `"noUnusedParameters": true`: Flags unused parameters.
   - `"noFallthroughCasesInSwitch": true`: Prevents fallthrough in switch statements.

These configurations work together to create a robust development environment for frontend projects, ensuring code quality, consistency, and modern JavaScript support.
