## High-level description
This code defines two utility functions, `createChunkRequest` and `hybridSearchRequest`, that generate code snippets for making API requests to the Trieve.ai platform. These snippets are intended to be used as examples for developers integrating with the Trieve.ai API.

## Table of contents
- `createChunkRequest` function
- `hybridSearchRequest` function

## Symbols

### `createChunkRequest`
#### Description
Generates a JavaScript code snippet for making a `POST` request to the `/api/chunk` endpoint of the Trieve.ai API. This request is used to create a new chunk of data in a specified dataset.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| dataset | string | The ID of the dataset to add the chunk to. Defaults to a placeholder value. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| snippet | string | A JavaScript code snippet representing a `fetch` call to the Trieve.ai API. |

#### Internal Logic
The function constructs a string literal containing a `fetch` call with the following details:
- **Method:** `POST`
- **URL:** `https://api.trieve.ai/api/chunk`
- **Headers:**
    - `Content-Type`: `application/json`
    - `TR-Dataset`: The provided `dataset` ID
    - `Authorization`: A placeholder authorization token
- **Body:** A JSON object containing:
    - `chunk_html`: Example HTML content for the chunk
    - `link`: A URL associated with the chunk

### `hybridSearchRequest`
#### Description
Generates a JavaScript code snippet for making a `POST` request to the `/api/chunk/search` endpoint of the Trieve.ai API. This request performs a hybrid search on the specified dataset.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| dataset | string | The ID of the dataset to search within. Defaults to a placeholder value. |

#### Outputs
| Name | Type | Description |
|:-----|:-----|:------------|
| snippet | string | A JavaScript code snippet representing a `fetch` call to the Trieve.ai API. |

#### Internal Logic
The function constructs a string literal containing a `fetch` call with the following details:
- **Method:** `POST`
- **URL:** `https://api.trieve.ai/api/chunk/search`
- **Headers:**
    - `Content-Type`: `application/json`
    - `TR-Dataset`: The provided `dataset` ID
    - `Authorization`: A placeholder authorization token
- **Body:** A JSON object containing:
    - `content`: The search query
    - `search_type`: Set to `hybrid`
