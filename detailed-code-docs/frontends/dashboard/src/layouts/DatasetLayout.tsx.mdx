## High-level description
The `DatasetLayout` component defines the layout for dataset-specific pages in the Trieve dashboard. It includes a navigation bar, a sidebar, dataset-specific tabs, and renders the content of child routes within this layout.

## Table of contents
- DatasetLayout component

## Code Structure
The `DatasetLayout` component uses the `createEffect` hook to ensure the URL always includes a sub-page after the dataset ID. It then renders a layout consisting of a `ShowToasts` component, a `NavBar`, a `Sidebar`, and a main content area. The main content area displays the `DatasetName` and `DatasetTabs` components, followed by the content provided by child routes.

### Symbol Name
`DatasetLayout`

#### Description
This component defines the layout structure for pages related to a specific dataset in the Trieve dashboard. It ensures the URL includes a sub-page after the dataset ID and renders the layout with a navigation bar, sidebar, dataset-specific tabs, and the content of child routes.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| children | JSX.Element | The content to be rendered within the layout |

#### Outputs
This component renders the layout structure for dataset-specific pages.

#### Internal Logic
1. Uses `createEffect` to check the URL pathname on component load and when the location changes.
2. If the pathname has a length of 4 (indicating it's missing a sub-page after the dataset ID), it appends `/start` to the pathname using `navigate` from `@solidjs/router`.
3. Renders the layout structure including `ShowToasts`, `NavBar`, `Sidebar`, `DatasetName`, `DatasetTabs`, and the provided `children`.
