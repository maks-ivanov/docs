## High-level description
The `ContextWrapper` component serves as a container for wrapping child components with the application's global context providers. It specifically includes the `UserContextWrapper` and `DatasetContextWrapper`, making user and dataset information accessible throughout the application.

## Table of contents
- `ContextWrapper`

## Code Structure
The `ContextWrapper` component is a simple functional component that renders two context providers: `UserContextWrapper` and `DatasetContextWrapper`. The `DatasetContextWrapper` is nested within the `UserContextWrapper`, ensuring that the dataset context has access to the user context.

## Symbols

### Symbol Name: `ContextWrapper`
#### Description
This functional component acts as a wrapper to provide the application's global contexts using `UserContextWrapper` and `DatasetContextWrapper`.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| props | ContextWrapperProps | An object containing a single property `children` which represents the child components to be wrapped by the context providers. |

#### Outputs
The component returns a JSX fragment (`&lt;&gt;`) containing the nested context providers, `UserContextWrapper` and `DatasetContextWrapper`, with the provided `children` rendered within the `DatasetContextWrapper`. This effectively makes the context values accessible to all child components within this tree.
