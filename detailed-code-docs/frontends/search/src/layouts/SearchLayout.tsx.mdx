## High-level description
The `SearchLayout` component provides a basic layout structure for the search frontend application. It sets up the theme based on user preference or system settings, registers a service worker for offline functionality, and renders a navigation bar (`Navbar`) and a toast notification component (`ShowToast`).

## Symbols
### Symbol Name: `SearchLayout`
#### Description
This functional component defines the main layout for the search application. It handles theme settings, service worker registration, and renders the common elements of the application.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| props | { children: JSX.Element } | An object containing the children components to be rendered within the layout. |

#### Outputs
JSX.Element representing the rendered layout structure.

#### Internal Logic
1. **Theme Handling:**
   - Determines the preferred theme (light or dark) based on local storage, browser preference, or a default value.
   - Updates the `documentElement` class to apply the selected theme.
2. **Service Worker Registration:**
   - Checks if the browser supports service workers.
   - If supported, registers the service worker from '/sw.js'.
   - Logs the registration status to the console.
3. **Layout Rendering:**
   - Renders a `div` element with the base layout classes.
   - Renders the `Navbar` component.
   - Renders the `props.children` (content within the layout).
   - Renders the `ShowToast` component.

The code includes basic error handling for service worker registration, logging any errors to the console.
