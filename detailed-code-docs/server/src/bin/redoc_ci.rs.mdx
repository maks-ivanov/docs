## High-level description
This code defines a simple command-line application that prints the OpenAPI specification for the Trieve API to the console. It leverages the `utoipa` and `trieve_server` crates to generate the specification in JSON format.

## Table of contents
- `main` function

## References
- `trieve_server::ApiDoc`

## Symbols

### `main`
#### Description
This function generates and prints the OpenAPI specification for the Trieve API.

#### Inputs
None

#### Outputs
- Prints the OpenAPI specification in JSON format to the console.

#### Internal Logic
1. Calls `ApiDoc::openapi()` to generate the OpenAPI specification.
2. Converts the specification to a pretty-printed JSON string using `to_pretty_json()`.
3. Prints the JSON string to the console using `println!()`.

## Side Effects
- Prints output to the console.

## Dependencies
- `trieve_server`: Provides the `ApiDoc` struct for generating the OpenAPI specification.
- `utoipa`: Used for generating the OpenAPI specification.

## Error Handling
- The `unwrap()` method is used to handle potential errors during JSON serialization. In a production setting, it would be advisable to handle these errors more gracefully.
