## High-level description
This YAML file contains configuration values for deploying Qdrant, a vector database, using Helm in a Kubernetes environment. It defines various aspects of the deployment, including container image, service configuration, persistence, security settings, and other Kubernetes-specific parameters.

## Table of contents
- Image configuration
- Service configuration
- Ingress settings
- Health check probes
- Resource allocation
- Security contexts
- Persistence configuration
- Cluster configuration
- Metrics and monitoring
- API key settings

## Symbols

### `replicaCount`
#### Description
Specifies the number of replicas for the Qdrant deployment.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| replicaCount | integer | Number of replicas |

### `image`
#### Description
Defines the container image settings for Qdrant.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| repository | string | Docker image repository |
| pullPolicy | string | Image pull policy |
| tag | string | Image tag |
| useUnprivilegedImage | boolean | Whether to use an unprivileged image |

### `service`
#### Description
Configures the Kubernetes service for Qdrant.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| type | string | Service type (e.g., ClusterIP) |
| ports | array | List of port configurations |

### `ingress`
#### Description
Defines ingress settings for external access to Qdrant.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| enabled | boolean | Whether ingress is enabled |
| hosts | array | List of ingress host configurations |

### `persistence`
#### Description
Configures persistent storage for Qdrant.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| accessModes | array | Storage access modes |
| size | string | Storage size |

### `config`
#### Description
Allows modification of Qdrant's configuration.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| cluster | object | Cluster-specific configurations |

### `apiKey`
#### Description
Configures the API key for authentication in Qdrant.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| apiKey | string or boolean | API key value or generation flag |

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| Kubernetes | Deployment platform |
| Helm | Package manager for Kubernetes |
| Qdrant | Vector database being deployed |

## Configuration
| Option | Type | Default | Description |
|:-------|:-----|:--------|:------------|
| replicaCount | integer | 4 | Number of Qdrant replicas |
| image.tag | string | "v1.10.1" | Qdrant image version |
| service.type | string | ClusterIP | Kubernetes service type |
| persistence.size | string | 200Gi | Size of persistent storage |
| apiKey | string/boolean | "qdrant_password" | API key for authentication |
