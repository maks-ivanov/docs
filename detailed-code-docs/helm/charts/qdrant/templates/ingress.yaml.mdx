## High-level description
This Helm chart template defines a Kubernetes Ingress resource for the Qdrant application. It configures routing rules and TLS settings based on the values provided in the Helm chart.

## Table of contents
- Ingress resource definition
- Metadata configuration
- Ingress spec configuration
- Host rules and path configurations
- TLS configuration

## Code Structure
The template uses conditional statements and loops to generate the Ingress resource based on the provided values. It references helper templates for generating labels and names.

## Symbols

### Ingress Resource
#### Description
Defines a Kubernetes Ingress resource to manage external access to the Qdrant service.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| .Values.ingress | Object | Contains all ingress-related configuration values |
| .Values.ingress.enabled | Boolean | Determines if the Ingress resource should be created |
| .Values.ingress.additionalLabels | Object | Additional labels to apply to the Ingress |
| .Values.ingress.annotations | Object | Annotations to apply to the Ingress |
| .Values.ingress.ingressClassName | String | The Ingress class to use |
| .Values.ingress.hosts | Array | List of host configurations |
| .Values.ingress.tls | Array | TLS configurations for the Ingress |

#### Internal Logic
1. Checks if ingress is enabled
2. Sets metadata including name, labels, and annotations
3. Configures ingress class if specified
4. Defines rules for each host, including paths and backend services
5. Configures TLS settings if provided

## Dependencies
