## High-level description
This YAML file defines a Kubernetes ServiceAccount resource for the Qdrant application. It creates a ServiceAccount with a dynamically generated name, optional annotations, and standard labels.

## Table of contents
- ServiceAccount definition
- Metadata
  - Name
  - Annotations (conditional)
  - Labels

## Code Structure
The file consists of a single Kubernetes resource definition for a ServiceAccount. It uses Helm templating to dynamically generate values and include common labels.

## Symbols

### ServiceAccount Resource
#### Description
Defines a Kubernetes ServiceAccount, which is an identity for processes running in a Pod to authenticate with the API server.

#### Inputs
| Name | Type | Description |
|:-----|:-----|:------------|
| .Values.serviceAccount.annotations | map | Optional annotations for the ServiceAccount |

#### Internal Logic
1. Sets the API version and kind for the ServiceAccount resource.
2. Generates the ServiceAccount name using the "qdrant.fullname" template.
3. Conditionally includes annotations if provided in the Helm values.
4. Includes standard labels using the "qdrant.labels" template.

## Dependencies
| Dependency | Purpose |
|:-----------|:--------|
| qdrant.fullname | Helm template for generating the full name of the Qdrant resource |
| qdrant.labels | Helm template for including standard labels |

## Configuration
