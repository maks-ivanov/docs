## Notes
1. This implementation is useful for testing or as a placeholder when credential storage is not required or not yet implemented.
2. All methods consistently return nil for error, indicating that no operation can fail in this no-op implementation.
3. The `Get` method returns `false` as its second return value, consistently indicating that no credential was found.
4. This implementation might be used in scenarios where credential management is optional, allowing the system to function without actual credential storage.
